## 13. License Quick Reference

| License | Commercial Use | Attribution Required | Modify/Remix | Share-Alike Required |
|---------|---------------|---------------------|-------------|---------------------|
| **CC0** | Yes | No | Yes | No |
| **CC-BY 3.0/4.0** | Yes | **Yes** | Yes | No |
| **CC-BY-SA 3.0/4.0** | Yes | **Yes** | Yes | **Yes** (derivatives must use same license) |
| **CC-BY-NC** | **No** | Yes | Yes | No |
| **MIT** | Yes | Yes (include license) | Yes | No |
| **GPL** | Yes | Yes | Yes | **Yes** (source must be available) |
| **OGA-BY 3.0** | Yes | **Yes** | Yes | No |

### Best Practices

1. **Always check the specific license** on each asset page before using it -- even within the same website, different assets may have different licenses.
2. **Keep a credits file** in your project listing every asset used, its author, license, and source URL.
3. **CC0 is safest** for hassle-free use (Kenney.nl is entirely CC0).
4. **CC-BY** just requires you to credit the author somewhere in your game (about screen, README, etc.).
5. **CC-BY-SA** requires your derivative work to also be shared under the same license -- be careful with this one for commercial projects.
6. **Avoid CC-BY-NC** if you plan to sell your game or monetize it in any way.
